# ON_v3 修复总结

## 概述

本修复解决了 ON_v2 指标设计中的关键科学方法论问题，确保评估结果真实反映假设的创新性。

---

## 修复详情

### 修复 1: HD 计算逻辑 (最严重)

**问题**: 使用最小差异而非最大差异
```python
# ON_v2 (错误)
HD = mean(sorted(dissimilarities)[:K])   # 取K个最小值

# ON_v3 (修复)
HD = mean(sorted(dissimilarities, reverse=True)[:K])  # 取K个最大值
```

**影响**: 假设与历史文献的真实差异性被系统性压低

**预期结果**: FIG-MAC 的 HD 从 ~0.4 提升至 ~0.7，正确反映其历史创新性

---

### 修复 2: CI 归一化方法

**问题**: 年份归一化引入时间偏差
```python
# ON_v2 (不稳定)
CI = citations / year_avg_citations[year]
# 2020年分母: 81.5, 2024年分母: 13.6 (6倍差异!)

# ON_v3 (稳定)
CI = citation_percentile(citations, year)  # [0, 1] 百分位排名
```

**影响**: 消除时间偏差，同一年份内的引用排名更公平

---

### 修复 3: 移除对数压缩

**问题**: 对数变换压缩高引用论文的影响
```python
# ON_v2 (压缩)
ON_raw = HD * ln(1 + CI) / CD
# CI=1.0 → ln(2)=0.69
# CI=10.0 → ln(11)=2.40 (仅3.5倍差异)

# ON_v3 (线性)
ON_raw = HD * CI / CD  # 尊重真实差异
```

**影响**: 高影响力工作获得应有的权重

---

### 修复 4: 增加 K 值

**变化**: K=5 → K=10

**原因**: 减少小样本方差，提高稳定性

---
