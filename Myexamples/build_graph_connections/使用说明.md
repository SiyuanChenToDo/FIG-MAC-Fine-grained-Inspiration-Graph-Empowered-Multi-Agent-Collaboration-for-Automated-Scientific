# 跨论文Paper-Solution关系人工标注数据生成脚本使用说明

## 📋 脚本功能概述

本脚本用于生成跨论文Paper-Solution关系的人工标注数据，具体功能包括：

1. **从向量数据库读取数据**：自动从FAISS向量存储中提取Paper的abstract、core_problem和Solution数据
2. **生成跨论文配对**：随机生成1550条确保来自不同论文的配对数据
3. **LLM初步分类**：调用Qwen模型进行INSPIRED/RELATED/NONE三分类
4. **智能任务分配**：自动分配数据给三位博士生，包括共同校验和独立校验
5. **生成标注文件**：输出便于中文处理的JSON格式文件

## 🚀 使用步骤

### 1. 环境准备

确保已安装必要的依赖：
```bash
pip install camel-ai numpy
```

确保向量数据库已正确建立：
- 路径：`Myexamples/vdb/camel_faiss_storage/`
- 需要的索引：
  - `paper/abstract/paper_abstract.index`
  - `paper/core_problem/paper_core_problem.index`
  - `solution/solution/solution_solution.index`

### 2. 运行脚本

```bash
cd Myexamples/build_graph_connections
python 人工标注数据生成.py
```

### 3. 等待执行

脚本会依次执行以下步骤：
- **步骤1**：初始化LLM和Embedding模型
- **步骤2**：加载向量存储（显示每个存储的向量数量）
- **步骤3**：随机抽样数据（100个paper + 2000个solution，包含embeddings）
- **步骤4**：相似度分布统计分析（自动推荐最优阈值）
- **步骤5**：生成跨论文配对（使用相似度初筛，从高质量候选池中抽取1550条）
- **步骤6**：批量调用LLM进行三分类（每50条保存检查点，支持断点续传）
- **步骤7**：智能分配标注任务
- **步骤8**：生成标注文件（每个博士生独立目录）

**预计运行时间**：约20-40分钟（取决于LLM API响应速度和统计分析）

## 📝 标注说明

### 关系类型定义

| 类型 | 代码 | 定义 | 示例 |
|------|------|------|------|
| **启发关系** | `INSPIRED` | 论文B的解决方案可以直接或间接应用于解决论文A的问题，提供新的思路、方法或技术路径 | A研究图像分类问题，B提出的注意力机制可直接应用 |
| **相关关系** | `RELATED` | 两者在研究领域、技术或问题上高度相似，但不能直接解决问题，属于同一范畴或相关领域 | A和B都研究自然语言处理，但解决不同的子问题 |
| **无关系** | `NONE` | 两者之间没有明显的启发或相关性 | A研究计算机视觉，B研究数据库优化 |

### 标注流程

#### 共同校验（50条）
1. 三位博士生**独立**标注同一批数据
2. 每人在对应的字段（`human_classification_A/B/C`）填写分类结果
3. 完成后对比结果，讨论不一致的情况

#### 精确校验（每人100条）
1. **仔细阅读**Paper A的abstract和core_problem
2. **仔细阅读**Paper B的solution
3. **深入分析**两者的技术关联和方法相似性
4. 在`human_classification`字段填写分类结果
5. 如有疑问，在`notes`字段记录

#### 大致校验（每人400条）
1. **快速浏览**Paper A和Paper B的内容
2. 根据关键词和领域知识做出判断
3. 在`human_classification`字段填写分类结果
4. 如遇到边界情况，可标记为"UNCERTAIN"

### 标注建议

✅ **推荐做法**：
- 先阅读LLM的分类和理由，作为参考
- 如果同意LLM的分类，可直接确认
- 如果不同意，需重新分析后给出自己的判断
- 记录困难样本和边界案例

❌ **避免**：
- 完全依赖LLM分类不加思考
- 对模糊样本随意分类
- 忽略技术细节只看关键词

## 🔧 配置修改

如需修改配置，编辑脚本中的`CONFIG`字典：

```python
CONFIG = {
    # 基本配置
    "TOTAL_SAMPLES": 1550,          # 总样本数
    "COMMON_SAMPLES": 50,           # 共同校验数
    "SAMPLES_PER_PERSON": 500,      # 每人总样本数
    "PRECISE_SAMPLES_PER_PERSON": 100,  # 精确校验数
    "ROUGH_SAMPLES_PER_PERSON": 400,    # 大致校验数
    "ANNOTATORS": ["博士生A", "博士生B", "博士生C"],
    "SEED": 42,                     # 随机种子（保证可复现）
    
    # 相似度初筛配置（新增）
    "USE_SIMILARITY_FILTER": True,  # 是否启用相似度初筛
    "AUTO_TUNE_THRESHOLD": True,    # 是否自动调优阈值（基于统计分析）
    "SIMILARITY_THRESHOLD": 0.3,    # 手动设置的阈值（AUTO_TUNE=False时使用）
    "SIMILARITY_WEIGHTS": {
        "abstract": 0.4,            # abstract的权重
        "core_problem": 0.6,        # core_problem的权重
    },
    "CANDIDATE_POOL_SIZE": 5000,    # 初筛后的候选池大小
    "SAMPLE_SIZE_FOR_STATS": 10000, # 用于统计分析的样本数量
}
```

**重要参数说明：**
- `USE_SIMILARITY_FILTER=True`：启用相似度初筛，提高INSPIRED/RELATED比例
- `AUTO_TUNE_THRESHOLD=True`：自动分析相似度分布并推荐最优阈值
- `SIMILARITY_WEIGHTS`：abstract和core_problem的权重（建议core_problem权重更高）

## ❓ 常见问题

### Q1: 脚本运行很慢怎么办？
**A**: LLM分类需要调用1550次API，预计需要15-30分钟。脚本已内置限流机制（每10次调用休息1秒）以避免触发API限制。

### Q2: 如何修改LLM的分类prompt？
**A**: 编辑脚本中的`LLM_EVALUATION_PROMPT`变量，根据需要调整判断标准和示例。

### Q3: 向量数据库中数据不足1550条怎么办？
**A**: 脚本会自动检测可用配对数量，如果不足会给出警告并使用所有可用配对。

### Q4: 如何确保配对是跨论文的？
**A**: 脚本在生成配对时会检查`paper_id`，只保留paper_id不同的配对。

### Q5: 生成的JSON文件如何转换为中文？
**A**: 文件已使用`ensure_ascii=False`参数保存，支持直接显示中文。可使用任何文本编辑器或JSON查看器打开。

### Q6: 如何导入Excel进行标注？
**A**: 可使用Python的`pandas`库将JSON转换为Excel：
```python
import pandas as pd
import json

with open('标注任务.json', 'r', encoding='utf-8') as f:
    data = json.load(f)

df = pd.DataFrame(data['samples'])
df.to_excel('标注任务.xlsx', index=False)
```

### Q7: 为什么使用相似度初筛？
**A**: 随机抽样会导致大量NONE关系（约95%），浪费LLM调用成本。相似度初筛可以：
- 提高INSPIRED/RELATED比例到40-60%
- 降低LLM调用成本
- 提高标注数据质量

### Q8: 如果程序中途崩溃怎么办？
**A**: 脚本支持断点续传：
- LLM分类每50条保存一次检查点
- 重新运行会自动从上次中断处继续
- 检查点文件位于`annotation_data/checkpoints/`

### Q9: 相似度阈值如何选择？
**A**: 建议使用`AUTO_TUNE_THRESHOLD=True`：
- 脚本会自动分析10000对样本的相似度分布
- 基于统计学原理推荐最优阈值
- 确保能获得足够的高质量配对

### Q10: 抽样的paper数量可以调整吗？
**A**: 可以在主函数中修改：
```python
paper_sample_size = 100  # 增加到200可提高覆盖率
solution_sample_size = 2000  # 增加到5000可提高候选池质量
```
