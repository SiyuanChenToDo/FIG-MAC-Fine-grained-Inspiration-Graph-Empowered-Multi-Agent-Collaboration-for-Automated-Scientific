# aggregate_metrics_to_excel.py vs recalculate_all_with_improvements.py

## 核心区别

### 1. aggregate_metrics_to_excel.py（原始批量评估脚本）

**功能**：
- ✅ 从报告目录读取所有报告文件
- ✅ 从头开始计算所有指标
- ✅ 生成全新的Excel文件
- ✅ 支持命令行参数（灵活配置）

**输入**：
- 报告目录路径（`--report-dir`）
- 报告文件模式（`--report-pattern`，默认`*.md`）
- 可选：起始/结束索引、限制数量

**输出**：
- 全新的Excel文件（`--output-excel`指定路径）
- 包含所有评估结果

**特点**：
- 🔄 **完整重新计算**：从头开始处理所有报告
- 📁 **目录驱动**：从文件系统读取报告
- ⚙️ **灵活配置**：支持多种命令行参数
- 🔧 **自动ON归一化**：自动应用ON归一化处理

**使用场景**：
- 首次批量评估
- 需要处理新的报告目录
- 需要灵活控制处理范围

**命令示例**：
```bash
python Myexamples/evaluation_system/aggregate_metrics_to_excel.py \
    --report-dir Scientific_Hypothesis_Reports \
    --output-excel Myexamples/evaluation_system/batch_results/ours/metrics_summary.xlsx
```

---

### 2. recalculate_all_with_improvements.py（增量更新脚本）

**功能**：
- ✅ 从现有Excel文件读取报告路径
- ✅ 只更新指标值，保留原有数据结构
- ✅ 输出到新文件（`metrics_summary_improved.xlsx`）
- ❌ 固定路径，无命令行参数

**输入**：
- 固定的Excel文件路径：`Myexamples/evaluation_system/batch_results/ours/metrics_summary.xlsx`
- 从Excel中读取`report_path`列

**输出**：
- 新文件：`metrics_summary_improved.xlsx`
- 保留原有列结构，只更新指标值

**特点**：
- 🔄 **增量更新**：基于现有Excel更新
- 📊 **保留结构**：保持原有列和顺序
- ⚠️ **固定配置**：路径硬编码在脚本中
- ⚠️ **ON归一化不完整**：需要后续处理

**使用场景**：
- 已有Excel文件，只需要更新指标
- 修复异常值或重新计算特定报告
- 保留原有数据结构

**命令示例**：
```bash
python recalculate_all_with_improvements.py
```
