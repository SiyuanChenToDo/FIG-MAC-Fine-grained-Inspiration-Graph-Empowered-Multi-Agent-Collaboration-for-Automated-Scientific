### 3. 目标论文的研究问题
- How can we identify the layers in an instruction-tuned LLM that encode general, task-agnostic representations shared across multiple NLP tasks?

### 4. 目标论文的方案/方法
- The paper addresses this by leveraging the Model-Oriented Sub-population and Spectral Analysis (MOSSA) framework combined with Center Kernel Alignment (CKA) similarity metrics. For each task, single-task control models are trained on task-specific data, while the instruction-tuned LLM (Llama2-SFT) is compared against these controls. By calculating CKA similarity between representations in each layer of Llama2-SFT and the corresponding control models, the authors identify that early layers (1–9) exhibit low similarity to task-specific controls, indicating general, shared representations. This is further validated by observing that these layers show consistent behavior across diverse task clusters, such as closed-book QA and natural language inference, which rely on general linguistic knowledge rather than domain-specific patterns.
- The paper identifies transition layers (10–15) by analyzing how CKA similarity between the instruction-tuned model and task-specific control models increases significantly compared to earlier layers. In these mid-layers, representations become highly aligned with task-specific controls for diverse task clusters, such as reading comprehension with commonsense reasoning and paraphrase detection. This alignment indicates that instruction tuning induces a structural shift in these layers, where general representations are transformed into task-specific ones. The authors further validate this by demonstrating that transition layers exhibit higher variance in representation complexity (measured by principal component analysis) and stronger correlations with task difficulty metrics (e.g., Flesch–Kincaid grade level) compared to shared layers.
- The refinement layers (16–32) are characterized by their ability to maintain high CKA similarity to task-specific control models while reducing variance across different task clusters. The paper shows that these layers further refine task-specific representations by increasing alignment with specialized controls for complex tasks like structured data-to-text generation and translation. This is achieved through continued adaptation during instruction tuning, where the model leverages task-specific patterns learned in earlier stages. The stability of refinement layers is confirmed by their consistent performance across both seen and unseen tasks, as well as their reduced sensitivity to dataset size variations compared to earlier layers.

- **研究问题 (RQ)**: How can bidirectional task interactions be modeled through iterative message passing to enhance joint learning of aspect term extraction and sentiment classification?
- **现有方案 (Sol)**: The paper introduces an iterative message passing mechanism that operates on shared latent representations across tasks. During each iteration, task-specific outputs (e.g., aspect term predictions from AE or sentiment scores from AS) are transformed into informative signals and injected into the shared latent space. These signals are then combined with the current latent representations and used to update all tasks simultaneously. Specifically, the message passing process involves:
1. Initialization: A shared encoder generates initial latent representations for each token.
2. Task-Specific Processing: Each task (AE, AS, DS, DD) processes the shared latent vectors using its own module (e.g., CNN layers with attention for AS).
3. Message Generation: Task outputs are converted into contextualized messages using a re-encoding network. For example, AS generates attention weights indicating sentiment-relevant tokens, while DS produces document-level sentiment scores.
4. Latent Update: The generated messages are aggregated and used to update the shared latent vectors for all tasks. This updated representation is then used in the next iteration.
5. Iteration: The process repeats for a predefined number of rounds (T), allowing information to propagate across tasks and refine predictions iteratively.
The framework ensures that each task benefits from the latest knowledge generated by other tasks, creating a dynamic feedback loop that strengthens task correlations and improves overall performance.

We address this problem by proposing a neural multi-task learning framework, LITE, which extracts representations of English to-do tasks with a multi-head attention mechanism on top of a pre-trained text encoder. To adapt representation models to to-do texts, we collect weak-supervision labels from semantically rich external resources (e.g., dynamic commonsense knowledge bases), following the principle that to-do tasks with similar intents have similar labels. ### 3. 目标论文的研究问题
- How can multi-head attention mechanisms be adapted to effectively capture task intent from short, under-specified to-do texts combined with list names?
- How can weakly supervised auxiliary tasks be designed to provide semantically meaningful supervision for learning intent-aware representations of to-do tasks?

### 4. 目标论文的方案/方法
- The LITE framework addresses this by introducing an intent extractor module that uses multi-head attention over token representations, enhanced with token type embeddings to distinguish between description tokens and list name tokens. The process involves:
1. Input Encoding: The to-do description and list name are concatenated and encoded using a pre-trained transformer-based text encoder (e.g., BERT or RoBERTa), producing contextualized token embeddings.
2. Token Type Embeddings: Separate embedding vectors are added to description tokens and list name tokens to signal their semantic roles. These embeddings are dynamically computed using hyperbolic tangent activation and concatenated with the original token representations.
3. Multi-Head Attention: For each attention head, query vectors are generated through a trainable transformation layer, and attention weights are computed based on the similarity between queries and key-value pairs derived from the token representations. This allows the model to focus on semantically critical components (e.g., action-related words in descriptions or intent-indicative list names).
4. Intent Vector Generation: Attention-weighted value vectors from all heads are concatenated to form a final intent vector, which serves as the task representation. This design ensures that both the description and list name contribute adaptively to the representation, even when the description is highly underspecified.
- LITE introduces three coordinated auxiliary tasks to generate weakly supervised signals for training:
1. Autocompletion Objective:
- Goal: Predict the full, semantically complete version of a short to-do description.
- Implementation: Uses list-based templates (e.g., "buy [description] at [shop]") to generate expanded versions of underspecified descriptions. The model is trained to reconstruct these expanded forms using a two-layer GRU decoder with cross-attention, encouraging it to infer missing contextual details.
2. Pre-Action and Goal Generation:
- Goal: Infer prerequisite actions (e.g., "need to go to the store") and goal-oriented intents (e.g., "want to complete the task") for each to-do.
- Implementation: Leverages the COMET model (a fine-tuned BART architecture) to generate these outputs. The original to-do description (from the autocompletion task) is used as input, and the model generates pre-action or goal sequences using a relation token ("xNeed" or "xIntent"). These outputs are then used as supervision signals during training.
3. Action Argument Prediction:
- Goal: Identify key frame elements (FEs) such as actors, objects, or locations that define the task.
- Implementation: Uses FrameNet to extract FEs from expanded to-do descriptions. Non-core FEs are weighted by TF-IDF to reduce noise, and the model is trained using GILE (a label-embedding method) to predict relevant FEs from a joint embedding space.
These tasks collectively provide diverse semantic signals that guide the model to learn robust intent representations, even when direct supervision is unavailable.
- The LITE framework employs a weighted multi-task learning (MTL) objective that balances contributions from the three auxiliary tasks during training. Key aspects include:
1. Task-Specific Loss Functions:
- Each auxiliary task has its own loss function:
- Autocompletion: Cross-entropy loss over generated tokens.
- Pre-Action/Goal Generation: Cross-entropy loss over generated sequences.
- Action Argument Prediction: Binary cross-entropy loss for each frame element (FE).
2. Dynamic Task Weighting:
- Task weights are adjusted based on the number of target labels in each sub-task (e.g., fewer labels for action argument prediction). This prevents dominant tasks from overshadowing others during training.
3. Shared Encoder and Task-Specific Heads:
- A single pre-trained text encoder is used for all tasks, while task-specific modules (e.g., the intent extractor, GRU decoder, or GILE classifier) are trained jointly. This encourages the encoder to learn representations that are useful across all tasks.
4. Domain Adaptation Integration:
- The framework can be extended with domain adaptation (DA) by pre-training the encoder on to-do data using masked language modeling, further improving performance on task-specific objectives.
This coordinated MTL design ensures that the model learns representations that are both semantically rich and adaptable to downstream task management applications.
